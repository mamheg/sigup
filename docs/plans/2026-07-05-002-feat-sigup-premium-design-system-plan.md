# feat: SiGup — премиальная дизайн-система и поэкранный редизайн

## Summary

Довести SiGup до вида, «как будто над платформой работали год»: практично, красиво, удобно. Темперамент — **гибрид: плотность каталога уровня Wildberries в премиум-обёртке уровня Linear/Airbnb/Stripe**, с сохранением зелёной идентичности (`#244D33`) и тёплой палитры.

Работа = сначала **дизайн-система с конкретными числами** (типо-шкала, спейсинг, нейтраль+бренд+акцент как роли, слоистые тени с зелёным подтоном, токены движения), затем **прокачка примитивов и системы движения**, затем **поэкранный редизайн** (каталог → главная → карточка → кабинет/админка/формы → афиша/объявления/о проекте → вход/регистрация → шапка/футер), и в конце **визуальная петля через Playwright MCP на проде** (скриншот каждого экрана desktop+mobile → сверка с эталоном → правка).

Этот план — **визуальный слой** поверх уже отгруженного фундамента (дизайн-токены U1, роутинг U2, языки U7 из плана 001). Он **заменяет** визуальные юниты плана 001 (полировка каталога, переверстка страниц) более детальной спецификацией. Бэкенд не трогаем (отдельная договорённость: БД будет на Postgres, позже).

**Не входит:** бэкенд/авторизация (отложены), оплата/чекаут, тёмная тема (токены закладываем совместимо, но экраны не делаем сейчас), генерация иллюстраций.

---

## Problem Frame

Текущий SiGup выглядит «сгенерированным ИИ»: разнобой скруглений и теней, ad-hoc hex в обоих регистрах, несуществующие Tailwind-классы, отсутствие единой типо-шкалы, спейсинг-ритма и системы движения. Часть экранов уже на новых токенах (U1), часть — на старом стиле; каталог — базовый. Нет ощущения «дорого» и «продумано».

Цель — не «сделать красиво разово», а **система**: каждое визуальное решение (размер шрифта, отступ, тень, тайминг анимации) взято из обоснованной шкалы и переиспользуется, экраны собираются из единой библиотеки компонентов, а качество проверяется скриншотами. Именно системность + микро-детали (concentric-радиусы, оптическое выравнивание, tabular-nums, scale-on-press, стаггер) создают ощущение «года работы».

**Визуальный тезис:** «Тёплый премиальный маркетплейс Северного Кавказа — воздух и типографика бутика, плотность и продукт-форвардность большого маркета, зелёно-песочная палитра с золотым акцентом.»

## Requirements

- **R1.** Единая **типо-шкала** (Cormorant display ≥24px, Inter ≤20px), с дисциплиной весов и трекинга; `text-balance`/`text-pretty`, `tabular-nums` для цен/чисел.
- **R2.** Единый **спейсинг-ритм** (база 4/8px) и **сетка каталога** 2→6 колонок по брейкпоинтам; выверенные max-width и вертикальный ритм секций.
- **R3.** **Цвет и элевация как система:** тёплая нейтральная шкала 50–950 + зелёный бренд + золотой акцент (только CTA/цена/бейджи); слоистые тени с зелёным подтоном в 3 тира; «тест снятой тени» (карточка читается без теней).
- **R4.** **Система движения:** токенизированные длительности/изинги/пружины (bounce:0 для UI), стаггер-вход, hover-lift с гардом `hover:hover` и `prefers-reduced-motion`, скелетоны загрузки.
- **R5.** **Прокачанные UI-примитивы:** concentric-радиусы, оптическое выравнивание иконок, image-outline `black/10`, scale-on-press `0.96`, хит-эрии ≥40px, focus-состояния (WCAG AA).
- **R6.** **Премиальная карточка товара** и полировка **каталога** (1:1 фото, clamp тайтла с фикс-высотой, акцент цены, рейтинг, бейджи, hover-смена фото + lift, скелетоны, липкие фильтры, сортировка, счётчик).
- **R7.** **Поэкранный редизайн** всех страниц под систему: главная, карточка, кабинет, админка, форма создания, афиша, объявления, о проекте, вход/регистрация, шапка/футер.
- **R8.** Все состояния каждого экрана: default / hover / focus / disabled / loading (skeleton) / empty / error.
- **R9.** **Адаптив** без горизонтального скролла; мобильные паттерны (фильтры-шит, меню).
- **R10.** **Визуальная приёмка** через Playwright MCP на проде: скриншот каждого экрана desktop(1280)+mobile(390), сверка с тезисом и эталонами, фикс всего найденного.

**Success criteria:** беглый взгляд на любой экран → «дорого и продуманно»; ни одного разнородного радиуса/тени/цвета; каталог ощущается как настоящий маркет; движение улучшает иерархию, а не отвлекает; чек-лист «не выглядит как ИИ» пройден; Playwright-прогон без визуальных дефектов.

## Key Technical Decisions

Все числа ниже — спецификация, которой следует исполнитель (источники в конце).

### KTD-1. Типо-шкала (Refactoring UI + Stripe + правило серифа)
- **Шкала (hand-tuned, px):** 12 / 14 / 16 / 20 / 24 / 30 / 36 / 48. Хранить как токены (`--text-*`). H1 `clamp(32px, 5vw, 48px)`.
- **Cormorant Garamond** — только display ≥24px (H1–H3, hero, заголовки секций, акцент цены на featured); weight 600; трекинг зажат: −0.02em на 36–48px, −0.01em на 24–30px.
- **Inter** — всё ≤20px; line-height 1.5 для body, weights 400/500/600; трекинг нормальный.
- `text-balance` на заголовках (≤6 строк), `text-pretty` на body/описаниях/подписях; `tabular-nums` на ценах/метриках/счётчиках.

### KTD-2. Спейсинг и сетка
- База 4px, основной шаг 8px (совпадает со шкалой Tailwind). Вертикальный ритм секций: 64–96px desktop, 32–48px mobile.
- **Каталог-грид:** 2 (≤480) → 3 (640) → 4 (1024) → 5 (1280) → 6 (1536) колонок; гэп 12–16px; 24–48 карточек с ленивой догрузкой.
- Max-width: каталог 1440–1536px, контентные страницы 1200px; боковые гэпы 16/24/32.

### KTD-3. Цвет и элевация
- **Нейтраль:** тёплая шкала 50–950 (подтон в песочный, не холодный серый), чтобы жить с `canvas #FBFAF7` / `surface #FFF` / `ink #211E1B`. Роли по образцу Radix 12-step: фоны (1–2), заливки интерактива (3–5), границы (6–8), сплошной бренд (9), hover (10), текст (11–12).
- **Бренд** зелёный `#244D33` (+ hover/soft/muted), **золото** `#C79E61` — акцент **только** для CTA/цены/бейджей, не для фонов и не градиентами.
- **Границы:** `line #ECE8E0` (1px) — разделители, плотные строки; тень для лифта, не для разделения.
- **Тени (Josh Comeau, зелёный подтон hsl бренда, не чёрный), 3 тира:**
  - `shadow-sm` — 1 слой, микролифт (инпуты/чипы);
  - `shadow-card` — 3 слоя (карточки, дропдауны);
  - `shadow-pop` — 5 слоёв (модалки, поповеры).
  «Тест снятой тени»: карточка остаётся организованной за счёт границы+спейсинга.

### KTD-4. Движение (Emil Kowalski)
- **Длительности:** press 120ms (`scale 0.96` — по правилу скилла, не ниже 0.95); tooltip 150; dropdown 200 ease-out; modal/drawer 300; page 250. Потолок UI ≤300ms.
- **Изинги:** ease-out `cubic-bezier(0.23,1,0.32,1)`; drawer/sheet `cubic-bezier(0.32,0.72,0,1)`. Пружины (motion): `bounce:0` для UI, `0.1–0.2` где уместна игривость.
- **Вход:** `opacity 0 + y 12 + blur 4 → 0`; **стаггер** 50–80ms, весь грид «оседает» <500ms; `AnimatePresence initial={false}` для дефолт-состояний.
- **Hover-lift карточки:** `translateY(-4px)` + углубление тени на 150–250ms ease-out; обёрнуто в `@media (hover:hover)`; уважать `prefers-reduced-motion`.
- **Скелетоны:** шиммер слева-направо, ~1.5s, прозрачность ~0.3, форма блоков повторяет контент.
- **Никаких** `transition: all`; только конкретные свойства; `will-change` точечно.

### KTD-5. Анатомия карточки товара (Baymard)
- Фото **1:1** (единое соотношение на весь грид), `object-cover`, image-outline `black/10`.
- Тайтл **2 строки** `line-clamp` с фикс min-height (выравнивание строк грида).
- **Цена** — тяжелее и крупнее тайтла, золотой/серифный акцент (второе, что сканируют после фото).
- **Рейтинг** — звёзды + число **на карточке** (не под hover), `tabular-nums`.
- **Бейджи** — верх-лево над фото, по флагу (скидка/новое/популярное), не всегда.
- **Hover** — смена на второе фото (200–300ms) + lift; на touch — без залипающего hover.

### KTD-6. Архитектура токенов и библиотека
- Всё в Tailwind v4 `@theme` (расширяем U1): `--text-*`, `--leading-*`, `--tracking-*`, `--shadow-*`, `--ease-*`, `--dur-*`, нейтраль/бренд/золото.
- Библиотека `src/components/ui/` (из U1) прокачивается до полного полиш-спека; движение — переиспользуемые варианты в `src/lib/motion.ts`.
- Гард стилей из U1 расширяем: бан ad-hoc hex в компонентах (только токены), контроль радиусов/теней.

### KTD-7. Референс-петля (что отличает «год работы»)
После каждого экрана — Playwright MCP на проде: скриншот desktop+mobile → сверка с тезисом и эталонами (Linear/Stripe/WB, Mobbin-паттерны) → правка. Один-два прохода на экран, не бесконечно.

## High-Level Technical Design

### Слои дизайн-системы (снизу вверх)

```
Токены (@theme)         тип · спейсинг · цвет/роли · тени · движение
   ↓
Примитивы (ui/)         Button Card Input Select Badge Modal Popover
   ↓                    + Skeleton Rating Chip Tabs Avatar Tooltip
Паттерны                ProductCard · FilterSidebar · SortControl ·
                        SectionHeader · EmptyState · PageHeader
   ↓
Экраны (pages/)         Home Catalog Detail Cabinet Admin Create
                        Afisha Announcements About Login Register
   ↓
Приёмка                 Playwright MCP screenshots (desktop+mobile)
```

### Инвентарь экранов и состояний (что должно существовать на каждом)

```
default → hover → focus → disabled → loading(skeleton) → empty → error
```

Каждый экран проходит все применимые состояния; каталог/списки обязаны иметь skeleton и empty; формы — error и disabled.

## Implementation Units

Нумерация продолжает план 001 (там U1–U11), чтобы не путать юниты в одной работе.

### U12. Токены дизайн-системы v2

**Goal.** Расширить `@theme` до полной спецификации (тип/спейсинг/цвет-роли/тени/движение) — фундамент, из которого собирается всё.
**Requirements.** R1, R2, R3, R4 (токены), R5.
**Dependencies.** U1.
**Files.** `src/index.css`, `docs/design/PRINCIPLES.md` (создать), `scripts/lint-styles.mjs` (расширить бан ad-hoc hex).
**Approach.** Добавить токены `--text-{xs..5xl}`, `--leading-*`, `--tracking-*`; тёплую нейтраль 50–950; роли бренда/золота; три слоистые тени с зелёным подтоном; `--ease-out`, `--ease-drawer`, `--dur-*`. Обновить `body` (antialiased, tabular по месту). `PRINCIPLES.md` — краткий живой свод (тезис, шкалы, do/don’t).
**Patterns to follow.** Refactoring UI (шкалы), Radix roles (цвет), Josh Comeau (тени).
**Test scenarios.**
- Covers R3. `npm run build` проходит; токены доступны как утилиты (`text-2xl`, `shadow-card`, `duration-[var]`).
- Гард падает на ad-hoc hex в компоненте и проходит на токене.
- Визуально (U21): один радиус/тень/цвет на всех новых экранах.
- `Test expectation: none` для чисто-декларативных токенов, поведение проверяется на экранах.

### U13. Прокачка UI-примитивов до полиш-спека

**Goal.** Довести `ui/` до уровня микро-деталей (скилл make-interfaces-feel-better).
**Requirements.** R5, R8.
**Dependencies.** U12.
**Files.** `src/components/ui/{Button,Card,Input,Select,Badge,Modal,Popover}.tsx`, добавить `Skeleton.tsx`, `Rating.tsx`, `Chip.tsx`, `Tabs.tsx`, `Tooltip.tsx`, `Avatar.tsx`.
**Approach.** Button: `scale-[0.96]` на active (проп `static`), оптическое выравнивание иконки (`pr = pl − 2px`), focus-ring. Card: concentric-радиусы (outer = inner + padding), тень-как-граница. Input/Select: focus AA. Badge/Chip: единые. Skeleton: шиммер. Rating: звёзды + `tabular-nums`. Хит-эрии ≥40px.
**Patterns to follow.** surfaces.md, animations.md (точные значения scale/blur/opacity).
**Test scenarios.**
- Covers R5. Кнопка визуально сжимается на нажатии; `static` отключает.
- Concentric: вложенная кнопка в карточке имеет радиус = внешний − padding.
- Скелетон рендерится и анимируется; форма повторяет карточку.
- Все интерактивные примитивы имеют видимый focus-ring.

### U14. Система движения

**Goal.** Единые переиспользуемые анимации вместо разрозненных inline-transition.
**Requirements.** R4.
**Dependencies.** U12.
**Files.** `src/lib/motion.ts` (создать), `src/components/layout/RootLayout.tsx` (page-переходы).
**Approach.** Экспортировать варианты: `staggerContainer`, `staggerItem` (opacity/y/blur), `hoverLift`, `pageTransition`, `iconSwap` (spring bounce:0). Гард `hover:hover` и `prefers-reduced-motion`. Переходы страниц по `location.pathname` через `AnimatePresence`.
**Patterns to follow.** Emil Kowalski motion table, animations.md.
**Test scenarios.**
- Covers R4. Грид входит стаггером, оседает <500ms; на повторном рендере без «прыжка» (`initial={false}` где нужно).
- `prefers-reduced-motion: reduce` → анимации отключены/минимальны.
- Переход между маршрутами плавный, без мигания.

### U15. Премиальная карточка + полировка каталога

**Goal.** Каталог, который ощущается как настоящий маркет (плотно + дорого).
**Requirements.** R6, R2, R8, R9.
**Dependencies.** U12, U13, U14.
**Files.** `src/components/catalog/{ProductCard,FilterSidebar,SortControl}.tsx` (создать), `src/pages/CatalogPage.tsx`.
**Approach.** ProductCard по KTD-5. Каталог: липкий сайдбар фильтров (desktop) / bottom-sheet (mobile), сортировка, живой счётчик, скелетоны при загрузке, empty-state, ленивая догрузка/пагинация, синхронизация фильтров с URL-query. Грид по KTD-2.
**Patterns to follow.** Baymard product-list, FilterSidebar паттерны.
**Test scenarios.**
- Covers R6. Карточка: фото 1:1, тайтл 2 строки фикс-высоты, цена акцентом, рейтинг виден без hover.
- Hover: смена фото + lift (desktop); на touch без залипания.
- Фильтр/сортировка меняют набор и счётчик; состояние в URL восстанавливается.
- Loading → skeleton-грид; пустой результат → empty-state со сбросом.
- Мобайл: фильтры открываются шитом, нет горизонтального скролла.

### U16. Редизайн главной

**Goal.** Первый экран = «постер», не дашборд; далее — ритмичные секции, ведущие в каталог.
**Requirements.** R7, R2, R8.
**Dependencies.** U12–U15.
**Files.** `src/pages/MainPage.tsx` (перенесён в pages в 001), `src/components/patterns/SectionHeader.tsx`.
**Approach.** Hero: один сильный визуальный якорь, бренд→заголовок(H1 serif, balance)→подзаголовок→CTA; поиск ведёт в `/catalog?q=`. Категории — единая система чипов/плиток → `/catalog?cat=`. «Популярное» — грид ProductCard (не лента из 5). Афиша/объявления — превью-секции со ссылкой на страницы. Стаггер-вход секций.
**Patterns to follow.** ce-frontend-design Module A (hero), SectionHeader.
**Test scenarios.**
- Covers R7. Hero читается как постер; один H1, один primary CTA над сгибом.
- Все кнопки главной ведут в валидные маршруты (`/catalog`, `/catalog?cat=`).
- Секции входят стаггером; на мобайле ритм/отступы сохранены.

### U17. Редизайн карточки товара

**Goal.** Детальная страница уровня премиум-PDP; убрать cheese-хардкод.
**Requirements.** R7, R8.
**Dependencies.** U12–U14.
**Files.** `src/pages/CardDetailPage.tsx`.
**Approach.** Галерея (соотношение, image-outline, превью), липкий блок контактов/цены на desktop, табы (О проекте/Товары/Доставка/Контакты) из данных проекта, «похожие» — той же категории (фикс бага 001), контакты скрываются при пустых полях (без junk-фолбэков), удалить мёртвый `mapZoom` и cheese-специфику.
**Test scenarios.**
- Covers R7. Контент берётся из данных проекта, не из хардкода.
- Похожие — той же категории; отсутствующие контакты скрыты.
- Галерея переключается; на мобайле липкий блок становится нижней панелью.

### U18. Редизайн кабинета, админки и формы создания

**Goal.** Дашборды/формы под систему + реальные данные.
**Requirements.** R7, R8.
**Dependencies.** U12–U14.
**Files.** `src/pages/{EntrepreneurCabinet,AdminPanel,CreateCardPage}.tsx`, `src/components/patterns/EmptyState.tsx`.
**Approach.** Метрики считаются из данных (не хардкод), `tabular-nums`. Cabinet: рабочая форма создания/редактирования (фикс мёртвой модалки), empty-state. Create: примитивы вместо inline-style, редирект+тост после отправки. Admin: реальные счётчики, рабочие разделы, approve/reject меняют статус. Dataviz-подход к метрикам.

**Test scenarios.**

## Risks & Mitigation

- **Скролл-крип (много экранов, легко разойтись).** Митиг* — сначала токены+примитивы (U12–U14), потом экраны только собирают из них; гард стилей ловит ad-hoc.
- **Переанимация.** Митигировать — движение только из `motion.ts`, потолок 300ms, `prefers-reduced-motion`, «удалить если чисто декоративно».
- **SSG-совместимость (план 001 U10).** Компоненты не трогают `window` вне эффектов; токены/CSS безопасны.
- **Задержка Vercel в петле приёмки.** Не открывать прод до окончания сборки; сверяться по хешу бандла/времени деплоя.
- **Серьёзная переверстка может задеть поведение.** Покомпонентные пуши + Playwright после каждого; менять визуал и явные баг-фиксы, не логику данных.

## Operational / Rollout Notes

- **Порядок:** U12 → U13 → U14 (система) → U15 (каталог, флагман) → U16–U19 (экраны) → U20 (сшивка) → U21 (петля приёмки, идёт на всём протяжении).
- **Цикл:** мелкие покомпонентные коммиты → push в `mamheg/sigup` (автор `mamheg`) → Vercel → Playwright MCP → фикс. Всегда есть точка отката.
- **Локальная проверка перед пушем:** `tsc --noEmit` + `lint:styles` + one-shot headless-скрин; прод-скрины — через MCP после деплоя.
